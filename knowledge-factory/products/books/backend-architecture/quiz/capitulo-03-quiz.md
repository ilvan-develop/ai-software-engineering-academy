# Quiz — Capítulo 3: Desenvolvimento Backend

**Instruções:** 5 perguntas de múltipla escolha. Apenas uma alternativa correta.

---

## Pergunta 1 (Fácil)

Qual decorator do NestJS é usado para definir um controller?

- [ ] A) `@Injectable()`
- [ ] B) `@Module()`
- [ ] C) **`@Controller()`**
- [ ] D) `@Component()`

**Resposta:** C

---

## Pergunta 2 (Fácil)

No NestJS, o que é um Guard?

- [ ] A) Um middleware que transforma dados de entrada
- [ ] B) **Uma classe que determina se uma requisição pode prosseguir**
- [ ] C) Um interceptor que modifica a resposta
- [ ] D) Um pipe que valida parâmetros

**Resposta:** B

---

## Pergunta 3 (Médio)

Qual a principal vantagem do NestJS sobre Express para aplicações Enterprise?

- [ ] A) É mais rápido em requisições por segundo
- [ ] B) **Possui injeção de dependência nativa e arquitetura modular**
- [ ] C) Não precisa de TypeScript
- [ ] D) É mais leve em termos de bundle size

**Resposta:** B

---

## Pergunta 4 (Médio)

O que um Pipe faz no NestJS?

- [ ] A) Intercepta requisições para logging
- [ ] B) **Transforma ou valida dados antes de chegarem ao handler**
- [ ] C) Protege rotas contra acesso não autorizado
- [ ] D) Gerencia conexões com banco de dados

**Resposta:** B

---

## Pergunta 5 (Difícil)

Qual padrão de design é usado pelo sistema de módulos do NestJS para organizar dependências?

- [ ] A) Singleton Pattern
- [ ] B) **Module Pattern com Injeção de Dependência**
- [ ] C) Factory Pattern puro
- [ ] D) Observer Pattern

**Resposta:** B
