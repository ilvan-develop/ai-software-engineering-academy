# Quiz — Capítulo 6: Design System

**Instruções:** 5 perguntas de múltipla escolha. Apenas uma alternativa correta.

---

## Pergunta 1 (Fácil)

Qual a diferença entre Style Guide e Design System?

- A) **Style Guide é um documento estático; Design System é um ecossistema vivo com componentes, tokens e processos**
- B) Style Guide é digital; Design System é impresso
- C) Style Guide é para designers; Design System é para devs
- D) Não há diferença, são sinônimos

**Resposta:** A

---

## Pergunta 2 (Fácil)

De acordo com o Atomic Design, qual a ordem correta dos níveis de complexidade?

- A) Moléculas → Átomos → Organismos → Templates → Páginas
- B) **Átomos → Moléculas → Organismos → Templates → Páginas**
- C) Átomos → Organismos → Moléculas → Templates → Páginas
- D) Páginas → Templates → Organismos → Moléculas → Átomos

**Resposta:** B

---

## Pergunta 3 (Médio)

O que são Design Tokens?

- A) **Variáveis visuais que abstraem decisões de design em valores reutilizáveis**
- B) Componentes React prontos para uso
- C) Logotipos e ícones do design system
- D) Arquivos de configuração do Storybook

**Resposta:** A

---

## Pergunta 4 (Médio)

Qual ferramenta é MAIS adequada para documentar componentes de um Design System?

- A) Figma
- B) **Storybook**
- C) VS Code
- D) Notion

**Resposta:** B

## Pergunta 5 (Difícil)

De acordo com o capítulo, qual o ROI aproximado de um Design System em uma organização com 5 squads?

- A) **R$ 3.8M/ano de economia líquida**
- B) R$ 100K/ano
- C) Não é possível calcular ROI de Design System
- D) O DS sempre aumenta custos no primeiro ano
