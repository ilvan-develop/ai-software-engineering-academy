# Certificação — Product Design

## Prova Objetiva (20 questões)

**Tempo estimado:** 60 minutos  
**Mínimo para aprovação:** 70% (14/20)  
**Pré-requisito:** Conclusão dos módulos 02 a 07

---

### Questão 1
O que é Product Discovery?
a) Uma técnica de prototipação rápida
b) Um processo estruturado para entender problemas e necessidades dos usuários antes de definir soluções
c) Uma metodologia ágil de desenvolvimento
d) Um framework de testes A/B

**Gabarito:** B

### Questão 2
Qual o objetivo do Design Thinking?
a) Criar designs visualmente atraentes
b) Resolver problemas complexos de forma centrada no humano, iterativa e colaborativa
c) Definir a arquitetura do sistema
d) Otimizar conversões de vendas

**Gabarito:** B

### Questão 3
O que diferencia UX (User Experience) de UI (User Interface)?
a) UX é visual, UI é funcional
b) UX abrange toda a experiência do usuário com o produto; UI é a camada visual e interativa
c) São sinônimos
d) UI é mais importante que UX

**Gabarito:** B

### Questão 4
O que são wireframes?
a) Protótipos funcionais completos
b) Representações de baixa fidelidade da estrutura e layout de uma interface
c) Código HTML/CSS da interface
d) Testes de usabilidade documentados

**Gabarito:** B

### Questão 5
Qual o propósito de um Design System?
a) Definir cores da marca
b) Fornecer um conjunto coeso de componentes, padrões e diretrizes para garantir consistência visual e funcional
c) Substituir prototipação
d) Automatizar testes de interface

**Gabarito:** B

### Questão 6
Qual técnica é usada para mapear a jornada do usuário?
a) User Story Mapping
b) Journey Mapping
c) Service Blueprint
d) Todas as anteriores

**Gabarito:** D

### Questão 7
O que é um MVP (Minimum Viable Product)?
a) O produto com o mínimo de funcionalidades possível
b) A versão mais simples do produto que entrega valor real e permite aprendizado validado
c) O protótipo de alta fidelidade
d) A versão final do produto

**Gabarito:** B

### Questão 8
Qual é a primeira fase do Design Thinking?
a) Prototipação
b) Teste
c) Imersão / Empatia
d) Implementação

**Gabarito:** C

### Questão 9
O que são personas?
a) Perfis de clientes reais
b) Representações semi-fictícias baseadas em pesquisa que sintetizam características, comportamentos e objetivos de usuários
c) Segmentos de mercado
d) Avatares visuais sem embasamento

**Gabarito:** B

### Questão 10
Qual a diferença entre validação qualitativa e quantitativa?
a) Qualitativa usa números, quantitativa usa entrevistas
b) Qualitativa busca entender o "porquê" (entrevistas, observação); quantitativa mede o "quanto" (dados, métricas)
c) Não há diferença
d) Quantitativa é mais importante

**Gabarito:** B

### Questão 11
O que é o Double Diamond?
a) Uma ferramenta de wireframing
b) Um modelo de processo de design com 4 fases: Descobrir, Definir, Desenvolver, Entregar
c) Um padrão de design visual
d) Uma métrica de UX

**Gabarito:** B

### Questão 12
Qual métrica mede a taxa de sucesso de uma tarefa em UX?
a) NPS
b) System Usability Scale (SUS)
c) Task Success Rate
d) Time on Task

**Gabarito:** C

### Questão 13
O que é acessibilidade em design?
a) Garantir que produtos possam ser usados por pessoas com diferentes capacidades e deficiências
b) Deixar o site bonito
c) Usar cores vibrantes
d) Adicionar animações

**Gabarito:** A

### Questão 14
Qual princípio de Gestalt afirma que elementos próximos são percebidos como grupo?
a) Similaridade
b) Proximidade
c) Continuidade
d) Fechamento

**Gabarito:** B

### Questão 15
O que é um design token?
a) Um prêmio de design
b) Uma variável que armazena valores de design (cor, espaçamento, tipografia) para uso consistente
c) Um componente de UI
d) Um framework CSS

**Gabarito:** B

### Questão 16
Qual a diferença entre protótipo de baixa e alta fidelidade?
a) Baixa fidelidade é mais rápida e explora conceitos; alta fidelidade se aproxima do produto final
b) Não há diferença significativa
c) Alta fidelidade dispensa testes
d) Baixa fidelidade é digital, alta é impressa

**Gabarito:** A

### Questão 17
O que é um teste de usabilidade?
a) Teste automatizado de interface
b) Observação de usuários reais realizando tarefas para identificar problemas de usabilidade
c) Revisão de código frontend
d) Teste de performance

**Gabarito:** B

### Questão 18
Qual framework prioriza features baseado em valor vs esforço?
a) RICE
b) MoSCoW
c) Value vs Effort Matrix
d) Todas as anteriores

**Gabarito:** D

### Questão 19
O que significa "design inclusivo"?
a) Design que inclui muitas cores
b) Design que considera a diversidade de usuários desde o início, não como adaptação posterior
c) Design acessível apenas para deficiências visuais
d) Design que funciona em todos os navegadores

**Gabarito:** B

### Questão 20
Qual é o papel de um Design System na escala de produtos?
a) Acelerar o time de design e desenvolvimento com componentes reutilizáveis e documentação
b) Substituir designers
c) Impedir mudanças de layout
d) Apenas padronizar cores

**Gabarito:** A

## Prova Prática — Projeto

**Título:** Redesign de um Fluxo Crítico

**Descrição:** Escolha um fluxo de um produto digital que você usa e:
1. Mapeie a jornada atual do usuário (as-is)
2. Identifique 3 problemas de usabilidade
3. Crie wireframes de baixa fidelidade para a solução proposta
4. Defina quais métricas você usaria para validar a melhoria

**Critérios:** Clareza da jornada (25%), Identificação de problemas (25%), Wireframes (30%), Métricas (20%)
