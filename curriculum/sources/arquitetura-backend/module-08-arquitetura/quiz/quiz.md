# Quiz — Módulo 08

## Pergunta 1

Qual princípio SOLID diz que uma classe deve ter apenas um motivo para mudar?

- a) Open/Closed
- b) Liskov Substitution
- c) Single Responsibility
- d) Dependency Inversion

**Resposta:** c

---

## Pergunta 2

Na Clean Architecture, as dependências de código fonte devem apontar:

- a) Para fora, em direção aos frameworks
- b) Para dentro, em direção ao centro (domínio)
- c) Em todas as direções livremente
- d) Para cima, em direção à interface do usuário

**Resposta:** b

---

## Pergunta 3

Em DDD, o que é um Value Object?

- a) Uma entidade com identidade única
- b) Um objeto imutável descrito por seus atributos
- c) Um serviço de domínio
- d) Um repositório

**Resposta:** b

---

## Pergunta 4

O que a letra "D" representa no SOLID?

- a) Dependency Inversion Principle
- b) Don't Repeat Yourself
- c) Data Integrity
- d) Dynamic Dispatch

**Resposta:** a
