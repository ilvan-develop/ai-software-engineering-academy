# Agentes para o Módulo 08

## Agentes envolvidos

| Agente | Função no módulo |
|--------|------------------|
| Curriculum Architect | Estruturar conceitos de Clean Arch, DDD, SOLID |
| Technical Writer | Escrever com exemplos práticos de código |
| Reviewer | Validar exemplos e consistência arquitetural |

## Instruções específicas

### Curriculum Architect

Ao planejar este módulo:
- Este é o módulo **mais denso** da formação — equilibrar profundidade com praticidade
- SOLID deve vir antes de Clean Architecture (fundação)
- Cada conceito deve vir acompanhado de código real (TypeScript)
- Conectar DDD com hexagonais com a estrutura de pastas que o aluno vai usar

### Technical Writer

Ao escrever este módulo:
- Máximo de exemplos de código — cada conceito deve ter um exemplo
- Usar a mesma stack da formação (NestJS + Prisma + TypeScript)
- Mostrar o "antes (errado)" e "depois (certo)" para cada conceito
- A seção de estrutura de pastas deve ser um guia prático, não teórico

### Reviewer

Ao revisar este módulo:
- Os exemplos de código estão corretos e seguem as boas práticas?
- A estrutura de pastas é consistente com Clean Architecture?
- As violações SOLID nos exemplos "ruins" são precisas?
- A progressão (SOLID → Clean Arch → DDD → Hexagonal → Prática) está lógica?
