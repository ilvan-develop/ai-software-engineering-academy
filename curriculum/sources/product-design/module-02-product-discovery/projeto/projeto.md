# Projeto Módulo 02 — Product Discovery de um SaaS Educacional

## Objetivo

Conduzir o processo de Product Discovery para uma **nova funcionalidade** de um SaaS educacional, gerando todos os artefatos de saída do discovery.

## Contexto

Você faz parte do time de produto de uma edtech chamada **Aprenda+**, um SaaS de cursos online voltado para empresas (B2B). O produto já existe e tem 500 empresas clientes, 15.000 usuários ativos.

### Cenário atual

A **Aprenda+** oferece:
- Cursos gravados em videoaulas
- Trilhas de aprendizado
- Certificados ao concluir cursos
- Dashboard do RH para acompanhar progresso dos funcionários

### O problema

O time de sucesso do cliente reporta que **empresas estão cancelando o contrato** porque "os funcionários não completam os cursos". A taxa de conclusão média é de **18%** (indústria: 30-40%). As empresas querem ver **maior engajamento e conclusão** dos treinamentos.

A diretoria definiu o seguinte **OKR** para o próximo semestre:

> **Objective:** Aumentar o valor percebido do Aprenda+ pelos clientes
> - **KR1:** Aumentar taxa de conclusão de cursos de 18% para 35%
> - **KR2:** Reduzir churn de 8% para 4% ao mês

Seu time foi escalado para fazer o **Product Discovery** de uma solução que ataque esse problema.

## Entregáveis

### 1. Pesquisa e Evidências

Conduza (teoricamente) as seguintes pesquisas e documente os resultados:

**a) Análise de dados:**
Com base nos dados abaixo, analise o funil de aprendizado e identifique onde está o maior drop-off:

```text
Home → Início do curso: 85%
Início curso → Aula 2: 60%
Aula 2 → Aula 5: 35%
Aula 5 → Conclusão: 18%
```text

- Qual etapa tem a maior queda relativa?
- Qual hipótese isso gera?

**b) Entrevistas:**
Crie um roteiro de entrevista semi-estruturada (8-10 perguntas) para:
- **Perfil 1:** Funcionário que não conclui cursos
- **Perfil 2:** RH / gestor que compra o Aprenda+

**c) Análise concorrencial:**
Pesquise (teoricamente) 3 concorrentes e identifique:
- Como eles resolvem o problema de engajamento?
- O que podemos aprender?
- O que NÃO copiar?

### 2. Frameworks

**a) JTBD:**
Defina o Job to be Done para:
- Funcionário que precisa fazer os cursos
- RH que contrata a plataforma

**b) Lean Canvas:**
Complete um Lean Canvas **para a nova funcionalidade** que você está propondo (não para o Aprenda+ inteiro).

**c) Value Proposition Canvas:**
Preencha os dois lados (Produto e Cliente) para a funcionalidade proposta.

### 3. Mapeamento

**a) Opportunity Solution Tree:**
Construa a árvore completa:
- Outcome (baseado no OKR)
- 3+ oportunidades
- 2+ soluções por oportunidade
- 1 experimento por solução

**b) User Story Mapping:**
Crie um User Story Map para a solução que você considera mais promissora:

- Eixo horizontal: etapas da jornada do usuário
- Eixo vertical: releases (MVP, R2, R3)
- Mínimo: 3 atividades, 2 releases

### 4. Hipóteses e Experimentos

**a)** Escreva **3 hipóteses** no formato:

```text
Acreditamos que [solução] para [público]
resultará em [outcome].
Saberemos que estamos certos quando [métrica/meta].
```text

**b)** Para cada hipótese, proponha um experimento (pode ser diferente — landing page, protótipo, teste A/B, concierge MVP) e detalhe:

- O que será testado
- Duração do experimento
- Métrica de sucesso
- Critério de go/no-go

### 5. Saídas do Discovery

Gere os seguintes artefatos:

**a)** **Proto-persona** para o perfil do funcionário e do RH

**b)** **3 user stories** priorizadas, com critérios de aceite, baseadas nas evidências coletadas

**c)** **Opportunity Backlog** com pelo menos 5 oportunidades, priorizadas (P0-P3), com evidência citada para cada uma

### 6. Apresentação para stakeholders

Escreva um **resumo executivo** (máximo 1 página) que você apresentaria para a diretoria, contendo:

1. Contexto do problema (com dados)
2. O que foi descoberto na pesquisa
3. A solução proposta (com justificativa)
4. O experimento para validar
5. O que será entregue se validado
6. Recursos necessários (tempo, pessoas)

## Critérios de avaliação

- [ ] Roteiro de entrevista bem estruturado (sem perguntas indutoras)
- [ ] Hipóteses no formato correto com métricas testáveis
- [ ] Opportunity Solution Tree completa (outcome → oportunidades → soluções → experimentos)
- [ ] User Story Mapping com corte de MVP claro
- [ ] Lean Canvas e Value Proposition Canvas preenchidos
- [ ] Proto-personas baseadas em evidência (não suposição)
- [ ] User stories com critérios de aceite
- [ ] Opportunity Backlog priorizado com evidência
- [ ] Resumo executivo convincente para stakeholders
- [ ] Experimentos com métricas de sucesso e critérios go/no-go
