# Quiz — Módulo 02

## Pergunta 1

Qual a principal diferença entre Discovery e Delivery?

- a) Discovery é feito por PMs, Delivery por devs
- b) Discovery decide **o que** construir; Delivery decide **como** construir
- c) Discovery é caro; Delivery é barato
- d) Discovery usa código; Delivery usa protótipos

**Resposta:** b

---

## Pergunta 2

Qual a ordem correta das fases do processo de Discovery?

- a) Ideação → Pesquisa → Oportunidade → Validação → Prototipação
- b) Oportunidade → Ideação → Pesquisa → Prototipação → Validação
- c) Oportunidade → Pesquisa → Ideação → Prototipação → Validação
- d) Pesquisa → Oportunidade → Prototipação → Ideação → Validação

**Resposta:** c

---

## Pergunta 3

Qual é a regra de ouro ao conduzir entrevistas com usuários?

- a) Falar 80% do tempo para guiar a conversa
- b) Perguntar "você usaria essa feature?" para validar a ideia
- c) Perguntar sobre comportamento passado, não intenção futura
- d) Sempre mostrar o protótipo antes de fazer perguntas

**Resposta:** c

## Pergunta 4

O framework JTBD (Jobs to be Done) parte de qual premissa?

- a) Usuários compram produtos com base no preço
- b) Usuários contratam produtos para fazer um trabalho
- c) Usuários escolhem produtos pela interface mais bonita
- d) Usuários preferem produtos com mais funcionalidades

## Pergunta 5

Qual framework é mais indicado para mapear o encaixe entre o produto e o cliente?

- a) Lean Canvas
- b) Opportunity Solution Tree
- c) Value Proposition Canvas
- d) User Story Mapping

## Pergunta 6

O que é uma Opportunity Solution Tree?

- a) Uma lista priorizada de funcionalidades

## Pergunta 7

Qual destas afirmações sobre MVP é verdadeira?

- a) MVP é a versão mais simples do produto final que entrega valor
- b) MVP é o menor experimento que valida ou invalida uma hipótese
- c) MVP deve ser lançado para todos os usuários

## Pergunta 8

No contexto Enterprise, qual prática é recomendada para fazer Discovery?

- a) Discovery acontece apenas no início do projeto
- b) Discovery e Delivery rodam em paralelo, com 20-30% do tempo para discovery
- c) Discovery deve ser terceirizado para consultorias
- d) Discovery só é necessário quando o stakeholder pede

## Pergunta 9
