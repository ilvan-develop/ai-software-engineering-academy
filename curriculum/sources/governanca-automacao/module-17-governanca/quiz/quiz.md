# Quiz — Módulo 17

## Pergunta 1

O que é Governança de TI?

- a) Um framework de desenvolvimento ágil
- b) Um sistema de regras, processos e práticas que alinha TI aos objetivos do negócio
- c) Uma ferramenta de monitoramento de servidores
- d) Um tipo de arquitetura de software

**Resposta:** b

---

## Pergunta 2

Qual a principal função do code review em um time de desenvolvimento?

- a) Encontrar todos os bugs antes de ir para produção
- b) Substituir a necessidade de testes automatizados
- c) Garantir qualidade, transferir conhecimento e manter padrões do time
- d) Apenas cumprir requisito burocrático

**Resposta:** c

---

## Pergunta 3

Qual das seguintes ferramentas é usada para garantir que commits sigam o padrão Conventional Commits?

- a) ESLint
- b) Prettier
- c) commitlint
- d) Dependabot

**Resposta:** c

## Pergunta 4

O que é um ADR (Architecture Decision Record)?

- a) Um relatório de bugs da aplicação
- b) Um documento que registra uma decisão arquitetural, seu contexto e consequências
- c) Um template de pull request
- d) Uma ferramenta de lint para arquitetura

**Resposta:** b

## Pergunta 5

Qual é a regra mais importante da gestão de segredos em governança de software?

- a) Usar senhas fortes
- b) Nunca commitar segredos no repositório
- c) Trocar senhas a cada 30 dias
- d) Usar apenas variáveis de ambiente

**Resposta:** b

## Pergunta 6

Qual a diferença entre SLO e SLA?

- a) SLO é para frontend, SLA é para backend
- b) SLO é uma meta interna baseada em SLIs; SLA é um compromisso contratual com o cliente
- c) SLO e SLA são a mesma coisa
- d) SLO é medido em segundos, SLA em horas

## Pergunta 7

Segundo a LGPD, o que o time de desenvolvimento deve fazer ao lidar com dados pessoais?

- a) Logar todos os dados para auditoria
- b) Nunca armazenar dados pessoais em banco de dados
- c) Não logar dados sensíveis, permitir exclusão real de dados, registrar consentimento

## Pergunta 8

Em uma branch strategy Trunk-based, qual é a característica principal?

- a) Branches de feature duram semanas
- b) Releases são feitas a cada 6 meses
- c) Branches são curtas (horas/dias) e o merge é frequente na main

## Pergunta 10

Qual é a abordagem recomendada para implementar governança em um time sem criar burocracia excessiva?

- a) Implementar todos os processos de uma vez para "já começar certo"
- b) Automatizar o máximo possível, começar pequeno, documentar o porquê e revisar periodicamente

**Resposta:** b
