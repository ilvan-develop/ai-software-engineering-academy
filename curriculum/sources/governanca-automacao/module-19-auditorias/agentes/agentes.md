# Agentes para o Módulo 19

## Agentes envolvidos

| Agente | Função no módulo |
|--------|------------------|
| Curriculum Architect | Estruturar os 16 tipos de auditoria |
| Technical Writer | Explicar o sistema de score e cada tipo |
| Reviewer | Validar exemplos de relatórios e scores |
| Auditor Agent | Referência prática — o agente que executa as auditorias |

## Instruções específicas

### Curriculum Architect

Ao planejar este módulo:
- Este é o módulo **mais exclusivo** da formação — tratar com a devida importância
- Conectar cada auditoria a um problema real que o aluno já enfrentou
- Mostrar que auditoria é um processo contínuo, não um evento único
- Enfatizar que o sistema de score permite comparação objetiva

### Technical Writer

Ao escrever este módulo:
- Tom de "diferencial competitivo" — este conteúdo não se encontra em outros cursos
- Incluir exemplos reais de relatórios de auditoria
- Mostrar o template universal que serve para qualquer tipo
- Enfatizar a classificação de riscos (Blocker a Minor)

### Reviewer

Ao revisar este módulo:
- O sistema de score é claro e reproduzível?
- Os exemplos de relatórios são realistas?
- A classificação de riscos está consistente?
- O ciclo de auditoria contínua faz sentido?

### Auditor Agent

Este módulo referencia diretamente o Auditor Agent da biblioteca de agentes. O agente deve ser capaz de:
- Executar cada um dos 16 tipos de auditoria
- Gerar relatórios no formato padronizado
- Atribuir scores consistentes
- Classificar riscos por gravidade
- Produzir planos de ação priorizados
