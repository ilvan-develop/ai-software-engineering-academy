# Checklist: Product Discovery Agent

## Problem Discovery
- [ ] Problema claramente definido (problem statement)
- [ ] Hipóteses documentadas
- [ ] Assumptions validadas com dados ou entrevistas
- [ ] Stakeholders identificados e consultados

## User Stories
- [ ] Formato: "Como [tipo], quero [ação], para [benefício]"
- [ ] Acceptance Criteria em Gherkin (Given/When/Then)
- [ ] Critérios de aceite são testáveis
- [ ] Histórias são independentes e negociáveis

## Priorização
- [ ] RICE score calculado (Reach, Impact, Confidence, Effort)
- [ ] Backlog priorizado por valor de negócio
- [ ] MVP claramente definido
- [ ] Dependências entre histórias mapeadas

## Validação
- [ ] Protótipo de baixa fidelidade criado
- [ ] Stakeholders validaram as histórias
- [ ] Critérios de sucesso definidos
- [ ] Métricas de acompanhamento definidas
